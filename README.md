# Cusal MARL Cryptocurrrency Forecasting Framework

## Аннотация
В настоящем проекте предлагается многоагентный подход к прогнозированию многомерных временных рядов, в котором каждая переменная рассматривается как отдельный агент в рамках actor-critic обучения с подкреплением. Архитектура явно интегрирует структурные причинные модели (SCM), задающие направленный ациклический граф зависимостей между переменными, что позволяет одновременно улучшать качество прогноза и обеспечивать интерпретируемость причинных связей.

## Введение

Стандартные модели прогнозирования временных рядов, как правило, фокусируются на статистической точности и неявно смешивают корреляционные и причинные зависимости. В условиях криптовалютного рынка, где индикаторы сильно взаимосвязаны, имеют лаговые эффекты и зависят от скрытых факторов (новости, ликвидность, манипуляции), такие модели часто дают нестабильные и неинтерпретируемые прогнозы.

Данная система вводит **структурное ограничение через причинно-следственный граф (SCM)**, где каждая переменная генерируется собственным причинным механизмом, зависящим только от её родителей в графе и собственной истории. Структура графа получена с помощью алгоритма **PCMCI**, что позволяет выделить истинные временные причинные зависимости между показателями.

Благодаря такому подходу агенты учатся прогнозировать не просто коррелированные значения, а **следствия причинных сигналов**, что критически важно для динамичных и взаимосвязанных рынков криптовалют.


## Метод

Пусть $x_t^{(v)}$ обозначает значение переменной $v$ в момент времени $t$. Тогда динамика переменной задается функцией:

$$
x_t^{(v)} = f_v(\{x_{t-\tau}^{(p)} : p \in Pa(v)\}, x_{t-k:t}),
$$

где $Pa(v)\$ — множество причинных родителей переменной $v$, $\tau$ — временной лаг, а $k$ — размер окна истории. Таким образом, информация о состоянии агента формируется из собственной истории переменной и историй ее причинных предшественников, что ограничивает поток информации структурой графа.

## Архитектура агентов

Каждой переменной соответствует отдельный RL-агент. Политика агента реализуется с помощью LSTM-энкодера, который обрабатывает временное окно наблюдений, после чего выдаются две головы:

- actor head для параметризации действия;
- critic head для оценки ценности состояния.

Действие моделируется гауссовым распределением:

$$
a_t \sim \mathcal{N}(\mu_t, \sigma_t),
$$

где $\mu_t$ и $\sigma_t\$ предсказываются политикой агента. Такая постановка согласуется с непрерывными действиями и actor-critic схемами.

## Обучение

Оптимизация проводится в рамках подхода PPO-style actor-critic. Функция потерь включает три компонента:

$$
\mathcal{L}_{actor} = - \mathbb{E}[\log \pi(a_t|s_t)\, A_t],
$$

$$
\mathcal{L}_{critic} = \mathbb{E}[(V(s_t) - R_t)^2],
$$

$$
\mathcal{L}_{entropy} = -\beta \cdot H(\pi).
$$

Итоговая функция потерь задается как:

$$
\mathcal{L} = \mathcal{L}_{actor} + 0.5\, \mathcal{L}_{critic} - 0.001\, \mathcal{L}_{entropy}.
$$

Для снижения дисперсии градиентной оценки используется Generalized Advantage Estimation (GAE):

$$
\delta_t = r_t + \gamma V_{t+1} - V_t,
$$

$$
A_t = \sum_{l=0}^{\infty} (\gamma \lambda)^l \delta_{t+l}.
$$

Данный механизм обеспечивает баланс между смещением и дисперсией при оценке преимуществ и является стандартным усилением policy-gradient методов.

## Среда

Среда `CausalEnv` формализует задачу прогноза как последовательность взаимодействий агента с временным рядом. На каждом шаге:

- состояние представляет собой скользящее временное окно;
- наблюдение включает собственную историю переменной и истории ее причинных родителей;
- действие соответствует прогнозу следующего значения;
- награда определяется как отрицательная ошибка прогноза:

$$
r_t = -|\hat{x}_t - x_t|.
$$

Тем самым задача прогнозирования преобразуется в задачу максимизации ожидаемой награды при сохранении каузальной интерпретируемости.

## Причинная значимость

Для количественной интерпретации влияния ребер используется эвристическая мера:

$$
I(p \rightarrow v) = \frac{|lag|}{1 + |Pa(v)|}.
$$

Эта величина отражает относительную силу причинного влияния с учетом временного лага и штрафа за высокую степень узла. Хотя такая метрика не заменяет полноценный каузальный анализ, она удобна как практический индикатор важности структурных связей.

## Тренировочный пайплайн

Процесс обучения включает следующие этапы:

1. Загрузка и предварительная обработка временных рядов.
2. Построение SCM-графа с фильтрацией по доступным переменным.
3. Инициализация политики для каждой переменной.
4. Выполнение rollout-эпизодов и сбор переходов.
5. Вычисление преимуществ с использованием GAE.
6. Обновление политик методом градиентного спуска.
7. Мониторинг награды, потерь, качества прогноза и причинной значимости.
8. Сохранение лучшей модели по выбранному критерию.

## Метрики

Система отслеживает следующие показатели:

- MSE и MAE для прогностического качества;
- суммарную награду и награду по агентам;
- actor loss и critic loss;
- динамику причинной важности рёбер;
- чекпоинты обученных политик.

## Запуск

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Подготовка данных

Положите временной ряд в `data.csv`, а описание причинной структуры в `scm.csv`.  
В `scm.csv` должны быть заданы узлы графа и направленные рёбра в output формате библиотеки [`tigramite`](https://github.com/jakobrunge/tigramite).

### 3. Запуск обучения

```bash
python main.py
```

### 4. Просмотр результатов

После обучения можно открыть результаты визуализации и метрики, которые сохраняются в папках проекта.  
Для интерактивного анализа дашбордов Streamlit выполните:

```bash
streamlit run streamlit_app.py
```

## Заключение

Предложенная система объединяет:

- интеграцию причинных графов в многоагентное RL;
- декомпозицию многомерного прогнозирования на независимые агенты;
- явное моделирование временных причинных зависимостей;
- интерпретируемость через оценки причинной важности;
- единую постановку задачи прогнозирования и анализа каузальной структуры.

Рассматриваемая архитектура формулирует прогнозирование временных рядов как многоагентную задачу обучения с подкреплением, в которой каждая переменная моделируется отдельной политикой, а причинные связи задаются структурной моделью SCM. Это позволяет совместить точность прогноза, ограничение пространства гипотез и интерпретируемость причинных воздействий в одной модели.
